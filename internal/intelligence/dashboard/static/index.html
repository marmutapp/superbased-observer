Compression savings

The proxy compresses each upstream request before forwarding — truncating large tool_result blocks and replacing dropped content with markers. The proxy can measure bytes directly; what costs you is tokens (estimated as bytes ÷ 4) and dollars (tokens × the row's input rate). Tokens and $ are the headline metrics; bytes are the source of truth.

Discovery — stale re-reads same-session only

Files re-read after they changed inside the same session burned prompt tokens to re-discover what the AI client should still have in context. The observer detects and reports these — it doesn't auto-prevent. The headline below is the savings opportunity if you fix the underlying habit (cache_control on hot files, shorter sessions, scoping context). Cross-session reads are excluded — a fresh session has no memory of a prior session, so re-reading a changed file is a first look, not a stale rediscovery.

Top files re-read

Repeated commands

Commands run multiple times with no relevant inputs changed in between — usually a sign the model didn't trust its own previous result. Each rerun pays output tokens for an answer that hasn't changed.

Cross-tool overlap multi-client

Files touched by more than one AI client (claude-code, cursor, codex, …) in the selected window. This is the visible side of cross-platform tool-call sharing: every MCP query (get_file_history, check_file_freshness, get_last_test_result, …) pulls from the same database, so when one agent reads a file, the next agent's query sees it. Click the heading for the full mechanism.

Patterns

Repeatable behaviours the observer noticed across your sessions — for example, "after running go test, you almost always run go vet", or "the file auth.ts shows up in 80% of sessions touching login.tsx". These get fed into observer suggest which writes them into CLAUDE.md / AGENTS.md / .cursorrules so new sessions inherit the habit without you re-typing instructions. Confidence is how sure the observer is the pattern is real — high-confidence patterns are documented behaviours, low-confidence ones are guesses.
